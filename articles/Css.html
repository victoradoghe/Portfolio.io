Modern Masonry Without Libraries

Grid, Columns, and Container Queries — When to Use Which

Overview

Modern masonry layouts mimic a natural stacking pattern, similar to Pinterest, where items of different heights stack closely without fixed rows. Instead of depending on JavaScript libraries like Masonry.js, modern CSS lets us build masonry natively using CSS Multi-Columns, CSS Grid, and Container Queries.

Why This Matters

Native CSS solutions improve performance, simplify code, and offer cleaner accessibility handling. This approach eliminates extra JS dependencies while delivering flexible modern UI layouts.

Multi-Column Layout

Best for: Pinterest-style feeds, image galleries, blogs.

Multi-Columns flow content top-to-bottom then left-to-right — like print columns.

.masonry {
  columns: 3;
  column-gap: 1rem;
}
.masonry-item {
  break-inside: avoid;
  margin-bottom: 1rem;
}

Pros: Simple, widely supported, no JS. Cons: Vertical reading flow (not ideal for card UIs).

CSS Grid — \"Dense\" Masonry

Best for: Card-based UIs, product layouts, dashboards.

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  grid-auto-flow: dense;
  gap: 1rem;
}

Pros: Maintains logical reading order and offers layout control. Cons: Not perfect masonry (dense = approximation).

True CSS Masonry (Experimental)

Modern browsers are adding real masonry layout to CSS Grid:

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  grid-template-rows: masonry;
  gap: 1rem;
}

Note: Experimental, needs fallbacks for production workflows.

Container Queries

Purpose: Make grids adapt to container size — not viewport size.

@container (min-width: 600px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

Use case: Responsive components inside flexible layouts.

When to Use Which

  • Columns → Pinterest-style media walls
  • Grid (dense) → UI cards, dashboards, product grids
  • Experimental Masonry Grid → Future-proof real masonry
  • Container Queries → Responsive reusable components

Conclusion

Use columns for simplest masonry, grid for structured control, and container queries for component-based responsiveness. Together, they replace JavaScript libraries and deliver highly performant and modern layouts.
